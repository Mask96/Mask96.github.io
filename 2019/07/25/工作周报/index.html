工作周报

发表于 2019-07-25 | 分类于 工作记录

工作日报

七月

第二周

2019.7.18 周四

  1. 完成定时内重复消息过滤（JudgeMessage.class）的单元测试。
  2. 完成定时任务类，使用quartz框架。可以使指定任务定时执行，也可以隔断时间执行。
  3. 完成mysql中两个表的消息查询，并设置定时任务，间隔为一天执行一次，每次执行时查询一天的数据，并且将重复的数据过滤掉。
  4. 钉钉发送消息有时间条数限制，获取了错误情况下返回值，并在发送消息过程中做了休眠操作。

2019.7.19 周五

  1. 完成了钉钉发送条数超限制的单元测试。
  2. 复查程序，补充logger。
  3. 修改了quartz中添加任务的逻辑。
  4. 项目整体复查并上测试环境。

周报

完成的工作：

  1. 为上周的RabbitMq接收信息实现了一个消息过滤机制（六个小时内重复消息不再发送，当消息超过100000条时清空有效时间外的信息）
  2. 使用jpa完成从mysql中获取24小时内计算出的商机信息，过滤掉重复的消息，并配置成为markdown发布到钉钉机器人中。
  3. 完成一个基于quartz的定时任务，在每日10点执行上面任务2。
  4. 完成大部分的单元测试，补充了logger，并发布到测试环境。

本周工作总结：

  1. 本周在完成mqsql部分内容时，学习了jpa来获取mysql中的数据的具体方法。
  2. 在完成定时发布时，学习了使用quartz来创建定时任务。
  3. 学习并且感受了优秀的单元测试实现和作用。

第三周

2019.7.22 周一

  1. 开始准备CMDB的前段开发工作
  2. 学习了HTML，CSS（内联元素，块元素，五种选择器，三种css布局）
  3. 初步学习了JavaScript的初级使用，如弹窗和初步的DOM，通过js获取或者改变页面中元素。

2019.7.23 周二

  1. 测试飞虎的rancher使用文档
  2. 排查mysql部分发送消息只有20条不再休眠后重新发送的bug，并做了一定的修补方案。
  3. 开会评估CMDB开发，培训git使用技巧。

2019.7.24 周三

  1. 进一步对js进行了学习，对BOM（浏览器对象模型）和DOM（文档对象模型，html划分为树结构，通过访问父节点、子节点、兄弟节点来操作标签）有了一定的学习和了解。
  2. 排查了devops-notify-center的日志，23日无问题。

2019.7.25 周四

  1. 完成hexo+next的个人博客搭建，建立了双ssh（公司gitlab和个人github）
